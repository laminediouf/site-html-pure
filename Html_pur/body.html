Corps d’un document HTML5

Après avoir détaillé l’en-tête situé entre les balise < head > et < / head >, intéressons-nous au corps de notre page web. Ce sujet étant si vaste que le reste de la page y sera dédiée.
Représentée entre les balises < body > et < / body >, vous placerez tout le contenu de votre page web à l’intérieur de celle-ci. Contrairement à l’en-tête, le corps du document s’affichera sur le navigateur de vos visiteurs.
L’élément < body > contient tous les contenus d’un document HTML5, comme du texte, des hyperliens, des images, tableaux, listes, etc …
Elle prend en charge tous les attributs standard et les attributs d’événement HTML5..

Évolution par rapport au HTML 4.01

En HTML5, tous les attributs spécifiques de la balise < body > sont supprimés, tandis qu’en HTML 4.01, ils étaient seulement déconseillées.

    • Elément ou balise Sens sémantique
    < h?>

    Définit des niveau d’importance de titre dans le corps du document
    Dans une page web, les titres introduisant le contenu dans l'HTML sont signalés par les balises h1, h2,
    h3, h4, h5 et h6 (h pour "heading" en anglais) et ont deux principales vocations :
    préparer la lecture du paragraphe que ces titres précèdent en résumant son contenu. Ils aident ainsi
    les lecteurs à mieux comprendre ce qu’ils vont lire.
    indiquer aux moteurs de recherche les informations les plus importantes que comportent votre page
    (l'algorithme de Google accorderait plus de poids aux mots clés et expressions que contiennent les
    titres qu'au reste du contenu)
    < p>

    La balise < p> définit un paragraphe.
    L’élément P crée automatiquement un espace avant et après lui.
    L’espace est appliqué automatiquement par le navigateur, ou vous pouvez le modifier dans une
    feuille de style CSS – margin (marge extérieur) et padding (marge intérieur).
    Attention à ne pas trop utilisé la balise < p>. Ne pas utiliser cette élément lorsque d’autres
    éléments sont plus appropriés (liste, définition …).
    exemple: < p> La balise P contient et affiche du texte < /p>

    < a >
    La balise A permet de définir un lien hypertexte vers un document en spécifiant l'url de ce
    document au niveau de l'attribut href.
    Le tag A peut également définir des ancres au sein d'un document et définir les liens pour y accéder.
    La page définie par le lien hypertexte s'affiche dans la page courante si l'attribut target n'est pas
    spécifié. Ce tag doit être fermé avec le tag < /A>.
    Il n'est pas possible de croiser ou d'insérer deux balises l'une dans l'autre.
    Ses attributs spécifiques sont :
    • href : document cible ouvert lors du clic sur le lien
    • hreflang :language du document
    • target :indique dans quelle page s'ouvre le document
    • title :texte affiché au survol du lien
    exemple:< a href=" " title="" target="">....< /a>
    < ul>

    Balise de fermeture obligatoire. < /ul>
    La balise < ul> est une liste non-ordonnée d'une page web xhtml (balise de type BLOC).
    Même si la balise < ul> est une balise de type BLOC, elle ne peut contenir que des balises < li>.
    exemple:
    < p>Voici une liste non-ordonnée :< /p>
    < ul>
    < li>Le XHTML est accessible< /li>
    < li>Le XHTML est bien structurer< /li>
    < li>Le XHTML donne un sens à vos page web :< /li>
    < ul>
    < li>C'est bon pour les internautes
    < li>C'est bon pour les moteurs de recherche< /li>
    < /ul>
    < div>

    Balise de fermeture oblgatoire : < /div>
    La balise div est un conteneur qui peut inclure tous les tags html (paragraphes, tables ...,
    et également d'autres divisions).
    Elle sert à structurer le document en plusieurs sections.
    Elle est l'alternative de la balise table pour ordonner les éléments dans une page et permet
    d'obtenir des présentations interressantes avec certains style CSS.
    exemple: < div style="width:300; height:180; overflow:auto; border:solid 1px black;"> < /div>

    < form>

    La balise < form> est utilisée pour créer un formulaire HTML pour la saisie des données
    utilisateur.Et donc la balise FORM déclare une zone de formulaire dans laquelle il sera possible
    pour les utilisateurs de saisir des informations qui pourront par la suite être récupérées côté
    serveur. Il est possible d'utiliser plusieurs formulaires dans une même page à condition de ne pas
    les imbriquer.Enfin il est impotant de donner une valeur à l'attribut name du formulaire pour y
    faire référence.Les structures utilisées à l'intérieur d'un formulaire pour échanger des
    informations sont les éléments :< input> , < textarea> , < button> , < select> , < option> ,
    < optgroup> , < fieldset> et , < label> ..
    L'élément de formulaire < form> ne doit contenir aucun < form> imbriqués.
    L'élément de formulaire < form> est de type block et donc il engendre un retour à la ligne avant et
    après l’élément.
    Les attribut pour la balise form:

    • accept:
      N’est pas pris en charge par HTML5.Spécifie les types de fichiers que le serveur accepte
      (et qui peuvent être soumises via un téléchargement de fichiers)
    • accept-charset:
      Spécifie le codage de caractères qui doivent être utilisés pour l'envoi du formulaire
    • action:
      Spécifie où envoyer les données du formulaire lors de la soumission d'un formulaire
    • enctype:
      Spécifie comment les données du formulaire doivent être encodées lors de la soumission
      d'une demande au serveur (uniquement pour la méthode = « post »)
    • method:
      Spécifie la méthode HTTP à utiliser lors de l'envoi des données de formulaire
    • name:
      Spécifie le nom d'un formulaire
    • novalidate:
      Spécifie que le formulaire ne doit être validé lorsque soumis
    Exemple: < form action="" method="get"> Nom: < input type="text" name="nom">
    Prénom: < input type="text" name="pnom">
    < input type="submit" > < /form>

    < button>

    La balise BUTTON permet de créer un contrôle de type bouton.
    C'est l'un des différents éléments utilisables au sein d'un formulaire FORM
    (au même titre que SELECT, TEXTAREA et INPUT).
    Il est possible de placer du code html à l'intérieur de l'élément button.
    Ses attributs spécifiques sont :
    - accesskey : défini un raccourci clavier simulant le clic
    - disabled : rend le bouton non cliquable (apparence grisée)
    - type : évènement associé au clic : submit, reset, button

    • Le type submit valide le formulaire dans lequel il est contenu. Il est équivalent à la fonction
      javascript submit().

    • Le type reset efface tous les champs renseignés dans le formulaire (la fonction javascript
      équivalente existe également).

    • NB: Toujours spécifier l'attribut type pour le bouton.

    Exemple:
    < button type="button" style="background:white; cursor:hand; border:solid 1px black;"
    style="background:white; cursor:hand; border:solid 1px black;"
    onclick="JavaScript: alert('Bouton Image')">
    < img src="../Images/image.gif">
    < /button>

    < img>

    La balise < img> insère une image dans un HTML page.
    Notez que les images ne sont techniquement pas insérés dans une page HTML, les images sont liées
    aux pages HTML.
    La balise < img> crée un espace dans la page HTML pour l'image référencée.
    Balise de fermeture en xHTML la balise doit être fermée comme ceci : < img />
    Les extensions les plus fréquentes sont .bmp, .gif et .jpg. Il est possible d'associer à une image
    un libellé sous forme de tooltip afin de la décrire au survol du curseur. La taille de l'image
    est également paramétrable et se fait de façon proportionnelle si une seule dimension est
    spécifiée.
    Ses attributs spécifiques sont :
    - src : Source de l'image, chemin absolu (http://...) ou relatif.
    - alt : Description de l'image
    - border : Détermine la bordure de l'image.
    - height / width : Défini les dimensions de l'image, utile pour mettre en forme la page durant le
    chargement des images.
    Exemple:
    < img src="../Images/image.gif" alt="..." border="..." width=".." height=".." />

    < input>

    La balise < input> spécifie un champ de saisie où l'utilisateur peut entrer des données.
    Les éléments < input> sont utilisés dans un élément < form> pour déclarer des contrôles d'entrée qui
    permettent aux utilisateurs d'entrer des données.
    Le champ d'entrée < input> a plusieurs variétés en fonction de l'attribut type voir
    le tableau en bas
    Exemple:
    Nom: < input type="text" name="nom">
    < nav>

    La balise < nav> définit un ensemble de liens de navigation.
    L'élément < nav> est destiné seulement pour les grands blocs de liens de navigation.
    Navigateurs, tels que les lecteurs d'écran pour les utilisateurs handicapés, peuvent utiliser cet
    élément pour déterminer s'il faut omettre le rendu initial de ce contenu.
    Exemple:
    < nav>
    < a href="/html/">HTML< /a>
    < a href="/css/">CSS < /a>
    < /nav>

    < table>

    La balise TABLE est un conteneur très utilisé; dans les pages web. Elle facilite la mise en forme des
    informations grâce à son découpage en lignes et cellules complètement paramétrables. Il est
    nécéssaire d'utiliser les balises TR et TD pour mettre en forme les éléments à afficher dans la
    table.
    D'autres balises peuvent structurer la table, voici toutes ces balises et leur ordre d'apparition :
    < TABLE>
    < CAPTION>
    < TR>
    < TH>
    < TD> < /TD>
    < /TH>
    < /TR>
    < /CAPTION>
    < /TABLE>
    < tr> < th> < td>

    La balise < tr> définit une rangée dans une table de HTML.
    La balise < th> définit une cellule d'en-tête dans une table de HTML.
    La balise < td> définit une cellule standard dans une table de HTML.

    • la balise TR est utilisée à l'intérieur de l'élément TABLE (balise de type BLOC). Elle
      détermine une nouvelle ligne dans celle-ci. Cette balise encadre les cellules TD(ou TH ) qui
      contiennent les informations qui seront affichées.
    • la balise TH est une cellule d'en-tête d'un tableau de données de la page web xhtml (balise de
      type en ligne).
    • la balise TD est utilisée au sein de la balise TR déterminant une ligne dans l'élément TABLE.
      Elle contient les données et informations à afficher (balise de type en ligne)
    Ses attributs spécifiques sont :
    - align : alignement du texte dans la cellule
    - valign : alignement vertical des éléments au sein des cellules
    - colspan : nombre de colonnes utilisées par la cellule
    - rowspan : nombre de lignes utilisées par la cellule
    - nowrap : empêche le retour à la ligne au sein de la cellule
    - background : Image de fond de la cellule
    - bgcolor : Couleur de fond de la cellule
    - bordercolor : Couleur des bordures de la ligne
    - bordercolordark : Couleur de la bordure sombre de la ligne (côté haut et côté gauche des
    cellules)
    - bordercolorlight : Couleur de la bordure claire de la ligne

    < select>

    La balise < select> est une liste déroulante d'une page web xhtml (balise de type EN LIGNE).
    La balise < select> s'utilise avec les balises < option> et < optgroup> (optionnelle).
    Cette balise n'est pas obligatoirement intégrée à un formulaire.
    La balise SELECT permet de créer une liste déroulante ou à choix multiples. C'est l'un des différents
    contrôles utilisables au sein d'un formulaire FORM pour rapatrier des informations
    (au même titre que BUTTON, TEXTAREA et INPUT). Elle encadre les valeurs de la liste qui sont
    définies dans les tags OPTION. Ses attributs spécifiques sont :
    - disabled : rend le contrôle passif, non modifiable.
    - multiple : autorise la sélection multiple d'éléments de la liste.
    - size : définit le nombre de valeurs visibles.
    L'évènement OnChange est également spécifique à la balise select. Il est déclenché lorsque
    l'utilisateur change la valeur sélectionnée dans la liste.
    Exemple:
    < select>
    < optgroup label="europe">
    < option>France< /option>
    < option>Angleterre< /option>
    < /optgroup>
    < optgroup label="afrique">
    < option>Algérie< /option>
    < option>Maroc< /option>
    < fieldset>

    La balise FIELDSET a été implémentée pour structurer les fomulaires.
    Elle permet de grouper de manière visuelle les champs qu'elle inclue.
    En utilisant la balise LEGEND qui lui est prop, il est possible d'attribuer un titre à ce groupe.
    Le but de la balise fieldset est de pouvoir découper un formulaire en catégories, pour le rendre
    plus lisible. Par défaut le tag fieldset pnd toute la largeur disponible. Pour limiter sa taille,
    il faut spécifier au niveau de l'attribut style la valeur de sa largeur avec l'élément width,
    ou en Utilisant le CSS. Exemple:
    < form>
    < fieldset>
    Prenom: < input type="text" size="30" />
    < /fieldset>
    < /form>
